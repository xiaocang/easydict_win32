# winfluent-rs 0701 Top 50 Fluent / Windows UI Audit

依据：

- Fluent UI / Fluent 2 设计体系：控件应由设计 token 驱动，覆盖颜色、排版、间距、状态、动效、层级和无障碍语义。
- Windows 应用 UI 框架通用标准：控件应有稳定的可访问名称、键盘与焦点行为、DPI 感知布局、窗口边界约束、系统主题和高对比度支持。
- 本清单只记录 `lib/winfluent-rs/` 库范围内的问题、修复和测试映射。

| ID | 问题 | 修复 | 测试覆盖 |
| --- | --- | --- | --- |
| WF-01 | 主题没有形成稳定的 Fluent token 覆盖面，控件容易继续散落硬编码颜色和尺寸。 | 已修复：`ThemeTokens::coverage_report` 量化颜色、排版、间距、状态、控件尺寸等 13 类 token。 | 测试覆盖：`visual_state_snapshot_and_theme_coverage_are_public_contracts`、`coverage_report_quantifies_design_tokens_used_by_controls`。 |
| WF-02 | Light / Dark / HighContrast / Minimal 模式缺少统一解析入口。 | 已修复：`ThemeTokens::resolve` 统一返回模式化 token。 | 测试覆盖：`resolves_visual_tokens_for_all_supported_modes`、`system_theme_resolves_to_light_visual_contract`。 |
| WF-03 | Body large 排版 token 偏离 Windows Fluent type ramp。 | 已修复：`Typography::default().body_large` 调整为 18px。 | 测试覆盖：`fluent_theme_tokens_follow_spacing_typography_touch_and_focus_contracts`。 |
| WF-04 | 间距缺少 4px 网格约束，容易出现非 Fluent 的零散布局。 | 已修复：`Spacing` 明确 xxs/xs/sm/md/lg/xl，保留 2px half-step 并约束其余步进。 | 测试覆盖：`fluent_theme_tokens_follow_spacing_typography_touch_and_focus_contracts`。 |

| WF-10 | 选中 tile / tab 状态没有独立颜色，容易只靠透明度表达选择。 | 已修复：新增 `selected_surface`、`selected_foreground`、`selected_border`。 | 测试覆盖：`selected_tile_renders_theme_selected_surface`、`tile_interaction_states_match_winui_static_tab_template`。 |
